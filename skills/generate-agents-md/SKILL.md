# スキル: AGENTS.md 生成

## 概要

プロジェクトを自動分析して `AGENTS.md` を生成するスキルの手順書です。

Claude Code では `/generate-agents-md` コマンドとして実行できます。

---

## インプット

| 項目 | 取得元 | 必須 |
|------|--------|------|
| ファイル構造 | `find` コマンド | ✅ |
| テックスタック | `package.json` / マニフェスト | 任意 |
| プロジェクト説明 | `README.md` | 任意 |
| 現在の実装状況 | ソースコードの有無 | ✅ |

---

## 分析ロジック

### テックスタック判定

| ファイル | スタック |
|---------|---------|
| `package.json` に `react` | React アプリ |
| `package.json` に `three` | Three.js 3D |
| `pyproject.toml` / `requirements.txt` | Python |
| `Cargo.toml` | Rust |
| `go.mod` | Go |
| `*.unity` / `Assets/` | Unity |

### 実装状態判定

- `src/` にファイルがある → 実装開始済み
- `src/` が空または存在しない → 仕様書段階
- `tests/` がある → テスト整備済み
- `dist/` または `build/` がある → ビルド可能

### プロジェクト略称（エージェント名）の決定

- プロジェクト名が2文字以上: 最初の3〜4文字を大文字に
- 例: `miro-like` → `@M3D`、`motion-agent` → `@MOTION`
- ユーザーが指定した場合はそちらを優先

---

## アウトプット品質チェック

生成した AGENTS.md が以下を満たすか確認すること:

- [ ] プレースホルダー `{{...}}` が残っていない
- [ ] 「現在の状態」がコードの実態と一致している
- [ ] 「次のステップ」が具体的（「実装」ではなく「XXXコンポーネントの実装」）
- [ ] 制約セクションに少なくとも1つの制約が記載されている
- [ ] 日付が今日の日付になっている
