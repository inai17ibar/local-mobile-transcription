# スキル: ADR 生成

## 概要

Architecture Decision Record（ADR）を `ADR.md` に対話式で追記するスキルの手順書です。

Claude Code では `/generate-adr` コマンドとして実行できます。

---

## ADR を書くべき判断基準

以下の場合に ADR を記録してください:

- 技術スタックの選定（フレームワーク、DB、ランタイム等）
- アーキテクチャの分割方針（モノリス vs マイクロサービス等）
- 外部サービスの採用・廃止
- パフォーマンスとシンプルさのトレードオフ
- セキュリティ上の制約
- 今後「なぜこうしたのか」を忘れそうな決定

以下は ADR 不要:

- 実装の詳細（変数名、関数の内部ロジック）
- リファクタリングで元に戻せる変更
- バグ修正

---

## ADR の状態遷移

```
提案中 → 承認済み → 廃止（新しい ADR に置き換えられた場合）
       ↘ 却下
```

---

## 番号付けルール

- ADR-001 から始める
- 番号は欠番なし・連番
- 廃止した ADR の番号は再利用しない

---

## AGENTS.md との連携

重要な制約を生む ADR（「〇〇は使ってはいけない」等）は、
決定後に `AGENTS.md` の「🚫 絶対守るべき制約」セクションに反映してください。

これにより AI エージェントが毎回 ADR.md を読まなくても制約を把握できます。
