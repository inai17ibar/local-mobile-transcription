# AGENTS.md 自動生成プロンプト

> **使い方**: 新しいプロジェクトで Claude Code を開き、このファイルの内容をチャットに貼り付けて実行してください。
> プロジェクトを分析して `AGENTS.md` を自動生成します。

---

このプロジェクトを分析して、`AGENTS.md` を生成してください。

## 手順

1. 以下のコマンドでファイル構造を確認してください:
   ```
   find . -maxdepth 4 -not -path './.git/*' -not -path './node_modules/*' -not -path './.venv/*' -type f | sort
   ```

2. 存在する場合は以下のファイルを読んでください:
   - `package.json` / `pyproject.toml` / `Cargo.toml` / `go.mod`（スタック・依存関係）
   - `README.md`（プロジェクト説明）
   - 既存の `AGENTS.md`（内容がある場合は上書きせず確認を取ること）

3. 以下を特定してください:
   - プロジェクト名・説明
   - テックスタック（フレームワーク、言語、主要ライブラリ）
   - テストアセット（サンプルデータ、フィクスチャ等）
   - 現在の状態（実装済み vs 未実装）
   - 次に着手すべきタスク

4. 以下の構造で `AGENTS.md` を生成してください:

```markdown
# AGENTS.md — {プロジェクト名} AI エージェント向けインデックス

> **このファイルは北極星（North Star）です。**
> AI ツール（Claude / Copilot / Cursor）はこのファイルを起点にプロジェクトを把握します。

> [!IMPORTANT]
> **{プロジェクトの現在フェーズを一文で}**
> {編集方針: 破壊的変更OK か 慎重に か}

---

## 📋 プロジェクト概要

**{プロジェクト名}** は、{一文の説明}

- **スタック**: {技術スタック}
- **テストアセット**: {テスト用ファイルがあれば}
- **対象**: {誰のためのプロジェクトか}

現在の状態：
- {実装済みの項目}
- **{次のタスク ← 次のタスクはここ}**

---

## 🔄 ADR 駆動設計

複雑なアーキテクチャ決定が必要な場合、以下のワークフローで進めます。

**ワークフロー**:
\`\`\`
1. [LANE:explore]   — @Explore: コンテキスト調査
2. [LANE:plan]      — @Plan: 複数アプローチを比較・提示
3. ★ ADR 検討       — ADR.md に候補・決定・根拠を記載
4. ✓ ADR 承認       — ユーザーと確認して決定
5. [LANE:implement] — @Implementer: 決定に基づき実装
6. [LANE:verify]    — @Verification: 動作確認
7. [LANE:review]    — @Reviewer: ADR 整合性チェック
\`\`\`

---

## 🤖 エージェント

| エージェント | 用途 | 呼び出し |
|------------|------|--------|
| `@{プロジェクト略称}` | 自律フルパイプライン | 高レベルタスク |
| `@Plan` | 設計・ADR 複数案提示 | アーキテクチャ決定が必要な際 |
| `@Explore` | 読み取り専用調査・Q&A | コード質問 |
| `@Implementer` | 承認済み計画の実行 | Plan 経由 |
| `@Reviewer` | 品質・セキュリティ・ADR 監査 | 実装後 |
| `@Verification` | lint / build / test 実行 | 実装後 |
```

5. `AGENTS.md` をプロジェクトルートに書き込んでください。

   - 空または1行の場合はそのまま上書きして構いません。
