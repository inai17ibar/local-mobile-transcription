ADR（Architecture Decision Record）の新しいエントリを `ADR.md` に追記してください。

## 手順

### Step 1: 現在の ADR を確認する

`ADR.md` を読んで、既存の ADR 番号を確認してください。
次の番号 = 最後の番号 + 1

### Step 2: ユーザーにコンテキストを確認する

ユーザーに以下を確認してください（まだ会話で明確でない場合）:

1. **どんな決定について記録しますか？** （例: DB の選択、フレームワークの選定）
2. **検討した選択肢は？** （複数あれば教えてください）
3. **最終的にどれを選びましたか？**
4. **その理由は？**

### Step 3: ADR エントリを生成する

以下のフォーマットで ADR エントリを生成し、`ADR.md` の「決定履歴」セクションに追記してください:

```markdown
## ADR-{番号}: {タイトル}

- **日付**: {今日の日付}
- **状態**: 承認済み
- **決定者**: {ユーザーが指定した場合はその名前、なければ「プロジェクトチーム」}

### コンテキスト
{なぜこの決定が必要になったか}

### 検討した選択肢

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| A: {選択肢A} | {メリット} | {デメリット} |
| B: {選択肢B} | {メリット} | {デメリット} |

### 決定
{選択した選択肢}: {決定の理由}

### 影響
- {この決定が与える影響}
- {トレードオフや今後の注意点}
```

### Step 4: AGENTS.md の制約セクションを更新する（必要な場合）

決定内容が制約に関わる場合（「〜を使ってはいけない」「〜は必須」等）、
`AGENTS.md` の「🚫 絶対守るべき制約」セクションに追記してください。
